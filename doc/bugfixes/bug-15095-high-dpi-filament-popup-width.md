## 问题现象
在高分辨率屏幕环境下调整系统缩放比例后，切换 `准备(Prepare)` 与 `预览(Preview)` 页面，右侧耗材区域显示会变小。继续点击耗材下半部分弹出的耗材编辑条时，条内的耗材编辑按钮显示不完整，影响耗材选择和编辑操作。

## 复现步骤
1. 在高分辨率屏幕环境下修改系统显示缩放比例。
2. 打开 Sanity Print，进入 `Prepare` 页面。
3. 在 `Prepare` 与 `Preview` 页面之间切换。
4. 点击右侧耗材区域的耗材下半部分。
5. 观察弹出的耗材编辑条。

## 禅道描述
（禅道原始描述）
高分辨在调整缩放比例后，切换准备与预览页面后，右侧区域的耗材排列显示过小，点击耗材展示的耗材编辑按钮不全。

## 根因分析
- 耗材下半部分点击后弹出的 `FilamentPopPanel` 宽度原先主要跟随父容器宽度计算，缺少稳定的最小宽度约束。
- 高分辨率和缩放比例变化后，弹窗内容会重新布局；当父容器宽度偏小时，弹出条会被压缩得过窄。
- 原逻辑里宽度和位置计算分散在点击弹出和缩放重算两处，维护上容易出现不一致，导致按钮区域显示不全。

## 修复方案
- 为耗材弹出条增加明确的最小宽度限制，避免在高 DPI/缩放场景下被压得过窄。
- 抽出统一的弹窗布局函数，集中处理弹出条的宽度、位置和重算逻辑。
- 在点击弹出和缩放重算时复用同一套逻辑，保证显示行为一致。

## 代码改动摘要
- `src/slic3r/GUI/FilamentPanel.cpp`
  - 新增 `layout_filament_popup(...)`，统一处理耗材弹出条布局。
  - 将耗材弹出条最小宽度设置为 `380`。
  - 点击耗材下半部分弹出条时，改为走统一布局逻辑。
  - 在 `msw_rescale()` 场景下，如果弹出条处于显示状态，也继续走统一布局逻辑。

- [ ] 在 `Prepare` 与 `Preview` 页面之间切换后，右侧耗材区域显示正常，不出现过小问题。
- [ ] 弹出条内的耗材选择框、喷嘴温度、热床温度、编辑按钮均完整显示。

## 风险与回退
- 风险：
  - 本次修改只调整耗材弹出条的最小宽度和布局收口逻辑，对其他区域影响较小。
  - 如果后续侧边栏整体宽度策略继续调整，弹出条最小宽度可能需要联动再评估。
- 回退：
  - 回退 `src/slic3r/GUI/FilamentPanel.cpp` 中关于 `layout_filament_popup(...)` 和最小宽度常量的修改。

## 备注
- 本次修复以“保证高 DPI/缩放切换后耗材弹出条完整显示”为目标，没有改变弹出方向和交互入口。
