## 2. 问题现象
- 准备页顶部工具栏中，当前项目/模型文件名称显示区域过短。
- 长文件名只能显示很少内容，用户无法直接从顶部栏识别当前文件。
- 初步放大文件名区域时，如果让它参与 toolbar 布局，会导致中间页签偏移，或在窗口还原时覆盖客服按钮、最小化、最大化、关闭按钮。

## 3. 禅道重现信息
- `[步骤]` 用户反馈：MichaelH。
- `[结果]` 准备页顶部文件名显示过短。
- `[期望]` 自适应展示文件名称。

## 4. 影响范围
- 模块: `Topbar 顶部工具栏`
- 关键文件:
  - `src/slic3r/GUI/BBLTopbar.cpp`
  - `src/slic3r/GUI/BBLTopbar.hpp`
- 受影响流程:
  - 准备页查看当前文件名称
  - 窗口拖拽缩放
  - 最大化与窗口化切换
  - 顶部页签与右侧系统按钮布局

## 5. 根因分析
- 原实现将文件名 Label 作为 `wxAuiToolBar` 的普通 control item 添加到工具栏中，并使用固定宽度 `TOPBAR_TITLE_WIDTH = FromDIP(80)`。
- 固定宽度过短，长文件名必然被过早截断。
- 直接增大该 control item 宽度会参与 toolbar 的 stretch spacer 分配，破坏“Online Models / Prepare / Preview / Device”页签居中。
- 最大化切窗口化时，toolbar item 的布局坐标存在短暂滞后，如果文件名按旧坐标刷新，可能覆盖右侧客服和窗口控制按钮。

## 6. 修复策略
- 保持顶部页签仍由原 toolbar 布局居中，不让文件名 Label 参与 toolbar 宽度分配。
- 将文件名 Label 改为 `BBLTopbar` 的浮动子控件，只在 `Device` 按钮右侧到客服按钮左侧分割线之间显示。
- 根据该空白区域动态计算文件名显示宽度，空间不足时使用省略号截断。
- 初始化、窗口宽度变化、最大化/窗口化切换后刷新文件名位置，并对布局尚未稳定的旧坐标做保护，避免覆盖右侧按钮。
- `SetLabel()` 后再次固定 Label 显示区域，避免 `wxStaticText` 首次设置文本时按文本自身宽度收缩，导致初始化位置不居中。

## 7. 代码改动摘要
- 文件: `src/slic3r/GUI/BBLTopbar.cpp`
  - 移除文件名 Label 的 `AddControl()` 布局方式，改为浮动子控件。
  - 保存客服按钮左侧分割线 item，用于计算文件名显示区域右边界。
  - 新增 `GetTitleDisplayRect()`，按 `Device` 页签右侧和客服分割线左侧计算显示矩形。
  - 新增 `LayoutTitleLabel()`，只在位置或尺寸变化时移动/缩放 Label，减少刷新抖动。
  - 新增延迟刷新调度，窗口状态切换后等待布局稳定再更新文件名位置。
  - 保留 tooltip 显示完整文件名。
- 文件: `src/slic3r/GUI/BBLTopbar.hpp`
  - 增加文件名显示区域计算、布局和延迟刷新相关成员声明。

## 9. 风险与回滚
- 风险等级: `低`
- 主要风险:
  - 顶部栏不同 DPI / 不同语言长度下的分割线坐标可能有差异，需要覆盖窗口缩放和最大化切换场景。
  - 文件名改为浮动子控件后，需要关注鼠标拖动标题栏区域的交互是否符合预期。
- 回滚方案:
  - 回滚 `BBLTopbar.cpp` / `BBLTopbar.hpp` 中本次文件名浮动显示区域相关修改，恢复原固定宽度 Label 显示逻辑。
