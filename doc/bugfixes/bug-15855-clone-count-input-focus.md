# Bug 修复记录

- 所属计划: `来源用例`
- 所属执行: `CP7.1.1 260430`
- Bug 类型: `代码错误`
- 严重程度: `一般`

## 2. 问题现象
- 在准备页选择模型并触发克隆功能后，克隆数量输入框没有自动获得键盘焦点。
- 焦点仍停留在模型颜色切换控件上，用户无法直接通过键盘输入克隆数量。
- 该行为削弱了克隆快捷入口的效率，用户仍需要额外点击数量输入框。

## 3. 禅道重现信息
- `[现状]` 选择克隆后，焦点没有移动到数量选择上，仍在模型颜色切换上。
- `[期望]` 选择克隆后，焦点移动到数量选择上。

## 4. 影响范围
- 模块: `模型操作 / 克隆`
- 关键文件:
  - `src/slic3r/GUI/GLCanvas3D.cpp`
  - `src/slic3r/GUI/GLCanvas3D.hpp`
- 受影响流程:
  - 选择模型后打开克隆数量设置浮窗
  - 通过键盘直接输入克隆数量
  - 使用克隆快捷入口快速复制模型

## 5. 根因分析
- 克隆数量控件原先使用 `ImGui::DragScalarV2`，其主要交互语义是拖拽调值，不是常驻文本输入框。
- `DragScalarV2` 只有在点击、导航激活等场景下才会进入临时文本输入状态，打开克隆浮窗时不能稳定承接键盘焦点。
- 克隆浮窗刚触发时，窗口和内部控件的 ImGui 状态需要经过后续渲染帧才稳定；如果仅在窗口首次出现帧设置焦点，首次进入克隆时焦点可能不会落到数量输入框。

## 6. 修复策略
- 将克隆数量输入区域改为使用 `ImGui::InputScalar`，使其成为真正的整数文本输入框。
- 保留 `draw_input_int_v2()` 后续的 `p_min / p_max` 范围限制逻辑，继续将克隆数量约束在合法范围内。
- 打开克隆浮窗时设置短帧数延迟标记，等待浮窗和控件布局稳定后，再在数量输入框绘制前调用 `ImGui::SetKeyboardFocusHere(0)`。
- 聚焦完成后将延迟标记复位为 `-1`，避免后续每帧抢占用户焦点。

## 7. 代码改动摘要
- 文件: `src/slic3r/GUI/GLCanvas3D.cpp`
  - 在 `draw_input_int_v2()` 中将 `ImGui::DragScalarV2` 替换为 `ImGui::InputScalar`。
  - 在克隆数量输入框绘制前增加延迟聚焦逻辑。
  - 触发 `ERenderEvent::ObjectCloneOptions` 时初始化延迟聚焦帧计数。
- 文件: `src/slic3r/GUI/GLCanvas3D.hpp`
  - 在 `CloneSettings` 中新增 `focus_input_on_open_frames`，用于记录打开克隆浮窗后的延迟聚焦状态。

- [ ] 选择模型后点击克隆，克隆数量输入框自动获得键盘焦点。
- [ ] 首次进入克隆浮窗时，可以直接键盘输入数量，无需额外点击输入框。

- [ ] 聚焦只在打开克隆浮窗时发生一次，不影响用户后续点击其它控件。

## 9. 风险与回滚
- 风险等级: `低`
- 主要风险:
  - `InputScalar` 与原 `DragScalarV2` 的交互语义不同，输入框区域不再承担拖拽调值行为。
  - 延迟聚焦帧数依赖当前浮窗渲染时序，如果后续克隆浮窗打开流程调整，需要重新验证首次聚焦。
- 回滚方案:
  - 回滚 `GLCanvas3D.cpp` 中 `InputScalar`、延迟聚焦逻辑和触发处帧计数初始化修改。
  - 回滚 `GLCanvas3D.hpp` 中 `CloneSettings::focus_input_on_open_frames` 字段。
