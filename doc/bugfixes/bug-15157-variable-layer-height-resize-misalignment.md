## 问题现象
在准备页面进入可变层高编辑状态后，拖拽缩放主窗口，右侧层高编辑条上的黑色基准线与蓝色层高曲线会相对背景条发生错位。缩放过程中蓝线还会出现弯曲形态异常，通常需要在编辑区域再次点击一次，线条位置才会恢复正常。

## 复现步骤
1. 打开 Sanity Print，进入 `Prepare` 页面。
2. 选中模型并进入可变层高编辑模式。
3. 在右侧显示层高编辑条时，拖拽主窗口边缘缩放窗口尺寸。
4. 观察右侧编辑条中的黑色基准线和蓝色层高曲线。

## 禅道描述
（禅道原始描述）
编辑可变层高时，缩放窗口，层高编辑基准线会显示错位。

## 根因分析
- 可变层高编辑条的背景、基准线和层高曲线都依赖画布尺寸计算几何数据。
- 原逻辑中，背景条在画布宽度变化时会重建；但基准线和层高曲线主要只在 `m_layer_height_profile` 变化时重建，窗口缩放时可能继续使用旧几何。
- 背景绘制发生在基准线/曲线绘制之前；如果背景阶段先更新了已缓存的画布尺寸，后续线条阶段会误判为“尺寸未变化”，从而跳过本次重建。
- 同时，旧逻辑只缓存了画布宽度，没有缓存画布高度；当窗口高度变化时，层高曲线的纵向映射也会失效，导致蓝线弯曲程度看起来被拉伸或压缩。

## 修复方案
- 为可变层高编辑图形缓存补充画布高度，确保宽高任一变化都能触发几何失效判断。
- 在背景条绘制阶段仅根据当前宽高判断是否重建，不提前写回缓存尺寸。
- 在基准线和层高曲线绘制阶段统一处理尺寸变化后的重建，并在重建完成时更新缓存尺寸。
- 将背景条和线条的条宽计算统一到 `thickness_bar_width(canvas)`，避免显示区域和线条坐标基准不一致。

## 代码改动摘要
- `src/slic3r/GUI/GLCanvas3D.hpp`
  - 在 `LayersEditing::Profile` 中新增 `old_canvas_height`，用于缓存上一次绘制使用的画布高度。
- `src/slic3r/GUI/GLCanvas3D.cpp`
  - 在 `render_active_object_annotations(...)` 中将背景条重建条件从仅判断宽度改为同时判断宽度和高度。
  - 在 `render_profile(...)` 中新增 `profile_geometry_changed`，在窗口宽高变化时强制重建黑色基准线与蓝色层高曲线。
  - 将基准线和层高曲线的横向坐标计算统一改为基于 `thickness_bar_width(canvas)`。
  - 将缓存尺寸更新放到线条几何重建阶段，避免背景条先更新缓存导致同帧线条跳过重建。

## 验证清单
- [ ] 进入可变层高编辑模式后，连续拖拽缩放主窗口，黑色基准线始终与右侧背景条对齐。
- [ ] 缩放过程中蓝色层高曲线始终贴合背景条显示，不出现横向漂移。
- [ ] 改变窗口高度时，蓝色层高曲线的纵向形态不再异常拉伸或压缩。
- [ ] 缩放结束后无需额外点击，基准线和层高曲线即可保持正确位置。
- [ ] 高 DPI 或系统缩放比例变化后，再进入可变层高编辑模式，显示行为仍然正常。

## 风险与回退
- 风险：
  - 本次修改集中在可变层高编辑条的 2D 叠加绘制缓存，不影响切层数据本身。
  - 若后续调整 `render_overlay()` 的调用顺序或编辑条宽度计算逻辑，需要同步检查背景条与曲线的缓存失效时机。
- 回退：
  - 回退 `src/slic3r/GUI/GLCanvas3D.hpp` 中 `old_canvas_height` 的新增字段。
  - 回退 `src/slic3r/GUI/GLCanvas3D.cpp` 中背景条和线条的宽高变化重建逻辑。

## 备注
- 本次修复目标是保证可变层高编辑条在窗口 resize 过程中的显示稳定性，不改变层高编辑算法和交互入口。
