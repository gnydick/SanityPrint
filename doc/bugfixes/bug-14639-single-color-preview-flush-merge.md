# Bug 修复记录

## 1. 基本信息
- Bug ID: `14639`
- 禅道链接: `https://zentao.creality.com/zentao/bug-view-14639.html`
- 标题: `预览页面，单色模型的长度和重量的计算总和上面显示的总量不一致`
- 首次处理日期: `2026-03-19`
- 文档更新日期: `2026-03-19`
- 所属产品: `Sanity Print`
- 所属模块: `切片预览`
- Bug 类型: `代码错误`

## 2. 问题现象
- 单色模型场景下，预览页面顶部总量与下方按类型拆分的长度、重量统计不一致。
- 在多色机型上打印单色模型时，预览中仍可能出现独立的 `Flushed` 展示，导致用户误以为存在额外冲刷耗材分类。
- 多盘场景下，如果每个盘分别都是单色模型，总览统计仍可能把冲刷单独列出，造成展示口径不统一。

## 3. 禅道重现信息
- `[前置条件]` 选择多色机型，导入单色模型，存在起始冲刷或相关冲刷统计。
- `[步骤]` 切片后查看预览页顶部总量、`Line Type` 与 `耗材丝` 统计。
- `[结果]` 长度/重量分项总和与顶部总量不一致，且单色模型场景仍显示独立冲刷项。
- `[期望]` 单色模型场景下不单独显示冲刷，展示口径与顶部总量一致。

## 4. 影响范围
- 模块:
  - `Line Type` 线型统计
  - `耗材丝 / ColorPrint` 统计
  - 多盘总览统计
- 关键文件:
  - `src/slic3r/GUI/GCodeViewer.cpp`
- 受影响流程:
  - 单色模型预览统计核对
  - 多色机型打印单色模型时的耗材展示
  - 多盘总览耗材统计展示

## 5. 根因分析
- 预览层对“单色模型中的冲刷”没有做特殊展示口径处理，仍将其作为独立 `Flushed` 项显示。
- 顶部总量与明细列表在业务语义上不一致：
  - 顶部总量关注最终总耗材；
  - 明细列表把单色模型中的冲刷继续拆成独立分类。
- 多盘总览最初按汇总结果判断是否单色，不足以覆盖“每个盘分别单色，但不同盘颜色不同”的场景。

## 6. 修复策略
- 对多色机型下的单色模型场景，统一展示口径：
  - `Line Type` 中将冲刷并入 `Custom`，不再单独显示 `Flushed`。
  - `耗材丝 / ColorPrint` 中将冲刷并入 `Model`，不再单独显示 `Flushed`。
- 对多盘总览统计改为“按盘判断”：
  - 每个盘先判断自己是否为单色模型；
  - 若该盘是单色模型，则把该盘冲刷量并入该盘的 `Model` 后再参与总览汇总。
- 保留底层冲刷统计数据，仅调整预览展示口径，减少对其他逻辑的影响。

## 7. 代码改动摘要
- 文件: `src/slic3r/GUI/GCodeViewer.cpp`
  - `FeatureType` 视图
    - 单色模型场景下，将冲刷时间/长度/重量并入 `Custom`。
    - 隐藏独立 `Flushed` 行。
  - `ColorPrint` 视图
    - 当满足“多色机型 + 单色模型”时，移除 `Flushed` 列显示。
    - 将冲刷长度、重量并入 `Model` 列与总计行。
  - 多盘总览统计
    - 改为按盘判断是否属于单色模型。
    - 对每个单色盘分别执行“冲刷并入 Model”的汇总逻辑。

## 9. 风险与回滚
- 风险等级: `低`
- 主要风险:

  - 多盘场景改为按盘判断后，需要关注“同盘多色、跨盘单色”的边界组合是否符合预期。
- 回滚方案:
